# 持仓做T匹配算法分析

**日期：** 2026-07-21
**主题：** 成交记录配对算法对比与推荐

---

## 一、问题定义

用户录入多笔买卖记录后，系统需要：
1. 计算当前持仓数量和成本
2. 识别每笔卖出对应哪笔买入（做T利润计算）
3. 支持正T（先买后卖）和反T（先卖后买）
4. 支持跨天做T和日内做T

核心矛盾：**一笔卖出可能对应多笔买入，一笔买入也可能被多笔卖出拆分。**

---

## 二、算法对比

### 1. FIFO（先进先出）

**原理：** 最早买入的持仓最先被卖出。

**示例：**
```
Day1: 买入 1000股 @ 10.00
Day2: 买入  500股 @ 11.00
Day3: 卖出  800股 @ 12.00
      → 卖出的是Day1的800股，成本 @ 10.00
      → 利润 = 800 × (12.00 - 10.00) = 1600
      → 剩余持仓：Day1剩200股 @ 10.00, Day2 500股 @ 11.00
Day4: 买入  300股 @  9.50
Day5: 卖出  600股 @ 13.00
      → 先卖Day1剩余200股 @ 10.00
      → 再卖Day2的400股 @ 11.00
      → 利润 = 200×(13-10) + 400×(13-11) = 600+800 = 1400
```

**优点：**
- 符合A股T+1规则本质（先买才能卖）
- 计算简单，易于理解
- 税务/财务上广泛采用
- 跨天做T自然支持

**缺点：**
- 无法精确追踪"某一笔做T"的独立利润
- 用户可能主观上觉得"我昨天买的今天卖"是一次做T，但FIFO可能配到了更早的底仓

**适用场景：** 底仓长期持有，偶尔做T降成本。这是最常见的散户模式。

---

### 2. LIFO（后进先出）

**原理：** 最后买入的最先被卖出。

**同上示例：**
```
Day5: 卖出 600股 @ 13.00
      → 先卖Day4的300股 @ 9.50
      → 再卖Day2的300股 @ 11.00
      → 利润 = 300×(13-9.50) + 300×(13-11) = 1050+600 = 1650
```

**优点：**
- 更贴合"日内做T"的心理模型（刚买的刚卖）
- 反T场景下更直观

**缺点：**
- 与A股T+1规则矛盾（当天买的当天不能卖）
- 财务上少用，可能造成税务问题
- 持仓成本波动大，不易理解

**适用场景：** 底仓不动，专门做日内T+0（需要有底仓支持）。

---

### 3. 加权平均法

**原理：** 每次买入后重新计算平均成本，卖出时按当前平均成本计算利润。

**示例：**
```
Day1: 买入 1000股 @ 10.00 → 持仓1000股, 成本10.00
Day2: 买入  500股 @ 11.00 → 持仓1500股, 成本10.33
Day3: 卖出  800股 @ 12.00 → 持仓700股, 成本10.33
      → 利润 = 800 × (12.00 - 10.33) = 1336
```

**优点：**
- 极其简单，一个公式搞定
- 持仓成本始终只有一个数字

**缺点：**
- **完全无法识别做T**
- 无法追踪单笔交易利润
- 多次做T后成本被"抹平"，失去分析价值
- 不满足用户需求

**适用场景：** 简单记账，不做T分析。pass。

---

### 4. 逐笔配对法（用户指定匹配）

**原理：** 用户每次卖出时手动指定"这笔卖出对应哪笔买入"。

**优点：**
- 最精确，完全符合用户主观意图
- 每笔做T利润清晰

**缺点：**
- 用户体验极差，每次卖出都要选
- 随着记录增多，选择列表越来越长
- 容易产生人为错误

**适用场景：** 专业交易员，交易频率极低。

---

### 5. 混合算法：智能FIFO + 日内优先匹配

**原理：**
1. **日内做T优先匹配**：同一交易日内，先买后卖的，优先互相配对
2. **跨天交易走FIFO**：日内的卖不完或跨天的，按FIFO匹配底仓
3. **反T特殊处理**：先卖后买的，卖出时标记为"融券"状态，后续买入时优先回补

**示例：**
```
Day1: 买入 1000股 @ 10.00（底仓）
Day2: 买入  500股 @ 11.00
      卖出  500股 @ 12.00（日内T）
      → 优先匹配：500股买入@11.00 对应 500股卖出@12.00
      → 利润 = 500 × (12-11) = 500
      → 底仓1000股不动
Day3: 卖出  600股 @ 13.00
      → 日内无买入，走FIFO
      → 卖出Day1底仓600股 @ 10.00
      → 利润 = 600 × (13-10) = 1800
      → 剩余底仓400股 @ 10.00
```

**优点：**
- 既符合T+1规则，又贴合做T心理模型
- 日内做T利润清晰独立
- 跨天交易自然处理
- 反T也能支持（先卖后买，卖出时标记待回补）

**缺点：**
- 实现复杂度最高
- 需要维护多层次的持仓队列

**适用场景：** 同时有底仓和做T习惯的用户。这是散户最常见的模式。

---

## 三、推荐方案

**采用"智能FIFO + 日内优先匹配"（方案5），原因：**

1. A股是T+1市场，日内做T本质上是"先买底仓的一部分，然后卖出底仓的另一部分"，但用户心理上觉得"我买了又卖了，是一次做T"
2. 如果全部走纯FIFO，用户昨天买的今天卖，系统可能配到了一个月前的底仓，用户体验差
3. 如果走LIFO，当天买的当天卖，虽然心理模型对了，但与T+1规则矛盾（当天买的不能当天卖，必须是已有底仓）
4. 混合算法最贴合实际：日内交易优先互配，跨天走底仓FIFO

## 五、关键计算逻辑

### 1. 录入买入
```
1. 检查 idempotent_key 是否已存在 → 存在则跳过
2. 创建 trade 记录，remaining = quantity
3. 无配对，直接加入持仓 lots
4. 重新计算 avg_cost = Σ(lot.cost × lot.quantity) / total_quantity
```

### 2. 录入卖出
```
1. 检查 idempotent_key 是否已存在 → 存在则跳过
2. 创建 trade 记录
3. 尝试日内匹配：
   - 查找同一天、时间更早的买入记录
   - 按 LIFO 匹配（最近的买入优先配最近的卖出）
   - 如果完全匹配 → 生成 closed 的 t_trade
4. 日内未完全匹配，走底仓 FIFO：
   - 按 trade 日期排序，找 remaining > 0 的买入
   - 逐个扣减 remaining，直到卖出数量满足
   - 每配对一次，生成一个 t_trade 或更新 cost
5. 如果卖出数量 > 总持仓 → 标记为"超卖/反T"，记录负持仓
```

### 3. 做T利润计算
```
正T利润 = sell_quantity × (sell_price - buy_price) - fees
cost_after = (原持仓总成本) / (原持仓数量)
           = (Σ(buy_price × quantity) - 做T利润) / (当前持仓数量)
```

## 六、边界情况处理

| 场景 | 处理方式 |
|------|----------|
| 部分成交 | 按实际成交数量录入，系统处理拆分 |
| 撤单 | 不录入，只录入最终成交 |
| 分红送股 | 手动录入"除权"调整记录，或从外部同步 |
| 反T超卖 | 允许负持仓，后续买入优先回补 |
| 多笔小单合成大单 | 按实际成交录入，匹配算法自动处理 |
| 跨天做T | 走FIFO底仓匹配，标记为跨天T |

## 七、结论

| 算法 | 复杂度 | 贴合T+1 | 做T分析 | 用户体验 | 推荐度 |
|------|--------|---------|---------|----------|--------|
| FIFO | 低 | ✅ | ⚠️ | 良 | ⭐⭐⭐ |
| LIFO | 低 | ❌ | ⚠️ | 良 | ⭐⭐ |
| 加权平均 | 极低 | ✅ | ❌ | 优 | ⭐ |
| 逐笔配对 | 中 | ✅ | ✅ | 差 | ⭐⭐ |
| **混合智能** | **高** | **✅** | **✅** | **优** | **⭐⭐⭐⭐⭐** |

**最终推荐：实现"智能FIFO + 日内优先匹配"**，这是唯一同时满足以下条件的方案：
1. 符合A股T+1交易规则
2. 精准识别每笔做T利润
3. 支持跨天和日内两种做T模式
4. 用户无需手动配对，系统自动处理
5. 成本变动追踪精确到每股
