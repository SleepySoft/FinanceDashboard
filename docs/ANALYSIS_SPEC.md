# 个股分析规范（Analysis Spec）

> **核心原则：分析是情报工作，不是数字罗列。**
> 
> 我们的目标是发现：这只股票为什么可能涨？为什么可能跌？有什么别人还没看到的东西？

---

## 一、分析前的准备

### 1.1 理解行业特性

不同行业有不同的核心矛盾：

| 行业类型 | 核心关注点 | 关键矛盾 |
|---------|-----------|---------|
| **银行/保险** | 资产质量、净息差、资本充足率 | 风险定价能力 vs 资产扩张冲动 |
| **消费品** | 品牌护城河、渠道控制力、同店增长 | 高端化能否持续 vs 消费降级冲击 |
| **制造业** | 产能利用率、订单能见度、技术迭代 | 规模效应 vs 产能过剩 |
| **科技/半导体** | 研发投入、产品周期、客户集中度 | 技术领先 vs 周期波动 |
| **公用事业/能源** | 特许经营权、成本传导、政策补贴 | 稳定现金流 vs 增长天花板 |
| **地产/基建** | 土地储备/订单、杠杆率、回款质量 | 高周转 vs 高负债 |
| **医药** | 管线深度、医保谈判、专利悬崖 | 创新回报 vs 政策压制 |

**基本要求**：分析报告必须体现对该行业核心矛盾的理解，不能跨行业套同一个框架。

### 1.2 理解公司特异性

每家公司有自己的"关键变量"：
- 比亚迪：电池成本下降曲线 + 出海进度
- 茅台：批价走势 + 渠道库存
- 宁德时代：产能利用率 + 海外客户拓展
- 招商银行：财富管理收入占比 + 零售存款成本

**基本要求**：先回答"这家公司的关键变量是什么"，再展开分析。

## 二、分析的核心要素

以下要素不要求全部覆盖，也不要求固定顺序。**根据股票特点，选择最有价值的3-5个维度深入展开。**

### 2.1 财务健康度

**不是罗列数字，而是回答：**
- 这家公司赚的钱是真的还是账面的？（现金流 vs 利润）
- 它的资产负债表有没有隐藏风险？（隐性负债、表外融资）
- 它的盈利质量在行业内处于什么水平？（ROE持续性、盈利波动性）

**关键指标**（按需选用）：
- 经营现金流/净利润 比值（判断盈利质量）
- 自由现金流（判断真实造血能力）
- 有息负债/经营现金流（判断偿债能力）
- 应收账款周转天数变化（判断行业话语权变化）

### 2.2 盈利能力与护城河

**核心问题：**
- 这家公司凭什么能比别人赚更多？这个优势能持续多久？
- 它的护城河是变宽了还是变窄了？

**分析角度：**
- 毛利率在行业中的位置及变化趋势
- 净利率与ROE的匹配度（高净利率+低周转 vs 低净利率+高周转）
- 成本结构变化（原材料、人工、折旧占比）

### 2.3 成长性判断

**核心问题：**
- 增长来自哪里？是行业红利还是公司自身能力？
- 增长的含金量如何？增收是否增利？
- 增长的可持续性如何？是一次性因素还是结构性变化？

**关键区分：**
- 内生增长（市占率提升、产品结构优化）vs 外延增长（并购、政策补贴）
- 量价齐升 vs 量增利减 vs 价增量减

### 2.4 行业格局与竞争态势

**核心问题：**
- 行业处于什么阶段？（导入期/成长期/成熟期/衰退期）
- 竞争格局是分散还是集中？趋势如何？
- 公司的竞争对手是谁？各自的优劣势？
- 行业有没有出现颠覆性变化？（技术替代、政策转向、新进入者）

### 2.5 估值与市场预期

**核心问题：**
- 当前估值反映了什么预期？这个预期合理吗？
- 估值与历史水平、同业相比处于什么位置？
- 市场预期差在哪里？（乐观预期落空的风险 / 悲观预期修复的空间）

**估值工具**（按需选用）：PE/PB/PS/DCF/分部估值，不强制要求具体方法。

### 2.6 催化剂与风险点

**催化剂**（可能推动股价上涨的因素）：
- 业绩超预期
- 新产品/新业务突破
- 行业政策利好
- 竞争格局改善
- 估值修复

**风险点**（可能推动股价下跌的因素）：
- 业绩不及预期
- 核心假设证伪
- 行业政策转向
- 竞争加剧
- 估值泡沫

**要求**：每个催化剂/风险点必须具体、可跟踪、有时间节点。

## 三、维度标签规则

维度标签（质/估/时/险/综合）是对分析的**综合判断**，不是对数字的机械映射。

### 3.1 质地（Quality）

评估公司的**长期竞争力**和**盈利质量**：

| 标签 | 含义 |
|------|------|
| 🟢 **green** | 护城河清晰、盈利质量高、管理层可信、行业地位稳固 |
| 🟡 **yellow** | 有一定优势但存在隐忧，或优势不够清晰 |
| 🔴 **red** | 护城河薄弱、盈利质量差、管理层有前科、行业地位边缘 |

**判断依据**：ROE稳定性、现金流质量、行业地位、管理层诚信记录、竞争优势可持续性。

### 3.2 估值（Valuation）

评估当前价格是否**合理反映了公司价值**：

| 标签 | 含义 |
|------|------|
| 🟢 **green** | 估值显著低于内在价值，安全边际充足 |
| 🟡 **yellow** | 估值合理，或存在分歧 |
| 🔴 **red** | 估值过高，或预期已经充分甚至过度反映 |

**判断依据**：相对估值（历史分位、同业比较）、绝对估值（DCF等）、市场预期差。

### 3.3 时机（Timing）

评估**当前是否是合适的介入/退出时机**：

| 标签 | 含义 |
|------|------|
| 🟢 **green** | 催化剂临近、技术形态支持、风险释放充分 |
| 🟡 **yellow** | 可跟踪但尚未到最佳时机 |
| 🔴 **red** | 风险尚未释放、催化剂不明确、技术形态不利 |

**判断依据**：催化剂时间表、技术面关键位置、风险事件进展。

### 3.4 风险（Risk）

评估**下行风险的大小和概率**：

| 标签 | 含义 |
|------|------|
| 🟢 **green** | 下行风险可控，最坏情况损失有限 |
| 🟡 **yellow** | 存在一定下行风险，需要跟踪 |
| 🔴 **red** | 下行风险大，可能出现永久性资本损失 |

**判断依据**：财务杠杆、经营杠杆、行业周期性、政策敏感性、黑天鹅概率。

### 3.5 综合判断（Verdict）

基于以上四个维度的**综合评估**，给出操作建议：

| 标签 | 含义 |
|------|------|
| 🟢 **green** | 值得重点关注，具备较好的风险收益比 |
| 🟡 **yellow** | 可跟踪，但需要更多信息或更好的时机 |
| 🔴 **red** | 当前不建议参与 |

**重要**：综合判断不是四个维度的简单平均。一个质地极好但估值极高的公司，综合可能是yellow；一个质地一般但极度低估且催化剂临近的公司，综合也可能是green。

## 四、写作原则

### 4.1 禁止清单

- ❌ **禁止模板化套话**："国内XX领域仍有进口替代空间"、"行业景气度与宏观经济高度相关"
- ❌ **禁止数据罗列不加分析**：列出5年营收数据却不说明趋势含义
- ❌ **禁止骑墙**："建议关注""需综合考虑"——必须给出明确判断
- ❌ **禁止跨行业套用同一框架**：银行分析制造业指标、制造业分析银行指标
- ❌ **禁止凑字数**：质量不以篇幅衡量，以洞察深度衡量

### 4.2 鼓励的做法

- ✅ **从问题出发**：先提出一个关键问题，再用数据回答它
- ✅ **对比分析**：与同行、与历史、与预期对比
- ✅ **情景分析**：乐观/中性/悲观情景下的回报与风险
- ✅ **可证伪的预测**：给出具体的预测和验证条件
- ✅ **承认不确定性**："这个判断基于XX假设，如果XX变化，结论会改变"

### 4.3 报告长度

不强制要求字数。有价值的分析可以很短，没价值的分析再长也是废话。

参考标准：
- **简要分析**：800-1500字，聚焦1-2个核心矛盾
- **深度分析**：2000-4000字，全面覆盖但有重点
- **超长报告**：仅在对公司有极深入了解时采用

## 五、输出标准

### 5.1 文件格式

```
data/<code>/reports/
  ├── fundamental_YYYYMMDD.md   (基本面分析)
  └── technical_YYYYMMDD.md     (技术面分析，可选)
```

### 5.2 报告头部

```markdown
# [股票名称]（[代码]）分析报告

**分析日期：** [YYYY-MM-DD]
**核心问题：** [一句话概括本次分析要回答的核心问题]
**情报来源：** [数据来源，如Tushare、公司公告、行业研报等]
```

### 5.3 维度标签

在报告末尾或meta.json中标注：

```json
{
  "quality": "green/yellow/red/none",
  "valuation": "green/yellow/red/none", 
  "timing": "green/yellow/red/none",
  "risk": "green/yellow/red/none",
  "verdict": "green/yellow/red/none"
}
```

**每个标签必须附带一句话理由**，说明为什么是这个判断。

## 六、自检清单

提交报告前自问：

- [ ] 我有没有发现什么别人可能没注意到的东西？
- [ ] 我的结论是基于数据还是基于模板？
- [ ] 如果我推荐买入，最坏情况会亏多少？
- [ ] 如果我推荐回避，错过这只股票的机会成本是什么？
- [ ] 我的预测是可证伪的吗？（什么情况下我会承认我错了？）
